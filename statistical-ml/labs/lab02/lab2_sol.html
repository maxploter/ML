Linear models

The aim of this lab is to learn to use R for fitting linear models, analyzing the fit and making predictions.

Fitting a linear model, interpreting output, visualising results

The commonly used command for fitting linear models is lm() command, although there are many other commands which can be used for the same task (glm(), for example). The first argument of the command is a formula, indicating the response variable and predictors used in modeling. It is usually of the form response ~ pred1 + pred2 +...+ predm, which means that in the case of numerical predictors we fit a function of the form \[\beta_0+\beta_1\cdot \mathrm{pred1}+\ldots+\beta_m\cdot \mathrm{predm}\] to data. The data frame where all the variables are, should be indicated with data=data_frame_name option and formula response~. means that all columns of the data frame (except column indicated as the response variable) are used for predictors. Some more advanced possibilities of specifying a more complicated formula are discussed later in the lab. Actually formulas are used for data preprocessing before applying a model fitting procedure. You can see the the effect of a formula to data by model.matrix(formula,data)

After fitting a model and saving the result into a variable (like model=lm(...)) we can see most important information about the model with summary(model)

Although it is possible to work with linear models by using base R, we are going to use some additional functions which help us to work with fitted models more easily. After loading the package broom (which was installed together with tidyverse and is loaded when we load tidymodels package) we have the following functions available: tidy(model) for viewing concise information about model coefficients, augment(model) to add predictions, residuals and so on to original data frame which was modelled and glance(model) to have an one line summary of model fit with various information like \(R^2\), residual standard error, AIC and so on.

If we want to predict values for new data, we should have a data frame with new data values for predictors we used for fitting model and use predict(model,newdata=data_for_prediction) command. For additional options (adding confidence/prediction intervals to predictions), look at help page of predict.lm.

Fitting a linear models by tidymodels framework

If we want to fit a linear model once for a concrete data set, then using lm command directly is probably the best option. Often we need to do the same thing repeatedly for model selection computations and in such cases it is better to use tidymodels framework, which unifies procedures for selecting, tuning and applying different models.

According to tidymodelsf framework we should define a recipe for data preprocessing (if no preprocessing takes place, then recipe just declares input and output variables), a model specification (which type of model we want to fit and which command we want to use for that purpose) and put those things together in a workflow. After that, we can fit the model, perform model tuning operations, compute various performance statistics and so on in a unified way. We are going to learn about different aspects of using this workflow during the course, but for current lab it is important to know that we can for a simple data preprocessor by rec=recipe(outcome_var~input_var1+...+imput_varm,data=dataset), we can specify that we want to use standard linear regression by model_spec=linear_reg() and that corresponding workflow is formed by wf=workflow()%>%add_model(model_spec)%>%add_recipe(rec). The model can be fitted by fitted_workflow=wf%>%fit(data=training_data), information can be obtained by tidy and glance, predictions can be obtained by predict command and augment allows one to add predictions to data. If we want to see more information about the fitted linear model, we can also extract the linear model fit by extract_fit_engine(fitted_workflow) and apply augment command to the linear model directly.

Detecting problems with model and/or data

Actually before starting to use a model for predictions and interpretation, one should make sure that there are no data errors which influence strongly the model, that the model is not biased and that the most important assumptions of linear models are satisfied. For this one can use several diagnostic plots.

  • Fitted values vs residuals (prediction errors). In ideal case the residuals should behave like random errors with the same variance for all levels of predicted values.

    • Bias means that we may have mostly positive errors for some predicted values (for example, for small predicted values or for large values of a predictor) and mostly positive errors for some other region of predicted values or predictor values. This means that we should add some computed terms to our model (or transform some predictors and/or the response). In the case of bias we can not trust statistics in the output since fitted model is quite different from the true model.
    • Heteroscedasticity means that the size of the residuals change with the predicted value. This also makes statistical information about coefficients invalid. Often can be corrected by using logarithmic (or more general Box-Cox) transformation of the response variable or by adding relevant variables to the model.

  • Predictors vs residuals. Help to detect prediction bias for some values of given predictor. Can be corrected by adding suitable computed variables.

  • quantile-quantile plot with respect to normal distribution. Measures of fit which use likelihood of observations are computed under assumption of normal distribution and may be invalid when actual distribution of residuals is quite different from the normal distribution. Additionally important in the case of small data sets

  • Plot of standardized residuals vs leverage. Residuals give information about outliers (suspicious values of the response variable). Leverage is a measure of how much a change in particular \(y\) value at given \(\mathbf{x}\) influences predicted vale \(\hat{y}\) (depends only on how different the current \(\mathbf{x}\) is from the average \(\mathbf{x}\) value and on the number of observations). High leverage indicates data points which may potentially have large influence on the fitted model. Combination of large (in absolute value) residual and large leverage indicates a point which influences model quite strongly. Cooks distance measures how much predictions change when a given point is removed from the data set and practical recommendation is that points with Cooks distance higher than 1 should be very carefully checked.

  • In order to add computed variables to our models, we may either add them to data frame before modeling or we can indicate computations inside formula. Use a recipe to add a variable inv_weight to the data and fit a model for mpg in terms of 1/weight to Auto data set (hint - you can remove weight variable in recipe by step_rm command). Is the fit better than before?

    Solution: Let us fit the model and look at residual plot again.

    rec_invweight=recipe(mpg~weight,data=Auto) |> step_mutate(inv_weight=1/weight) |> step_rm(weight)

mpg_invweight=wf_invweight |> fit(data=Auto)
ggplot(augment(extract_fit_engine(mpg_invweight)))+geom_point(aes(x=.fitted,y=.resid))+geom_smooth(aes(x=.fitted,y=.resid))
    ## `geom_smooth()` using method = 'loess' and formula 'y ~ x'

    Although there is some heteroscedacity still visible (errors for very small fited values seem to have smaller variance) and there may be some bias in the case of large values, the assumptions of fitting linear model are not strongly violated and the statistical information about the model may be trusted (at least by this diagnostic picture) We can compare the models by \(R^2\) statistic (since this does not depend on assumptions being valid):

    glance(mpg_weight)%>%bind_rows(glance(mpg_invweight))

    The second model describes large fraction of variance of the response variable, so it can be considered to be better.

Working with nominal/factor variables, adding interactions

Factor variables are nominal variables which are encoded by numbers. For example, we may have variable sex in our data set with values Female and Male and it becomes a factor variable if we encode the values by 0 and 1 (but also storing information about meaning of those numbers). Function lm works correctly with factor variables and also with textual variables (which it transforms to factors).

In case of fitting a model with a factor variable as predictor, actually for each level of the factor variable except one a new variable with values 0 and 1 is created and then usual linear model is fitted.

Interactions allow adding nonlinear effects to the model by considering products of variables. If we want to add to a model in addition to variables x and y the product x*y then we can indicate it in the model formula with response ~ x*y, so in case of product in model formula linear terms are automatically added. If we just want to add the product X*Y without linear terms, we can use I() or we can indicate it in the model formula with special operator : like response ~ x:y.

    • Compare the previous model to the model which contains also interactions of the two predictors used in the previous model. Which one would you use for predictions?

      Solution:

      rec4b=rec4|>step_dummy(Species)|>step_interact(terms=~starts_with("Species"):Petal.Length)
wf4b=workflow()|>add_model(mod4)|>add_recipe(rec4b)
wf4b_fitted=fit(wf4b,iris)
tidy(wf4b_fitted)
      glance(wf4b_fitted)

      In this case we have coefficient of Petal.Length also different for each species, so in fact we have fitted three separate models (one for each species). The interaction terms have relatively large p-values (definitely larger than 0.05), so we have statistically insignificant terms in our model. If our aim is making predictions, then our decision depends on which metric we decide to base it on. If we decide to use adjusted \(R^2\), then the model with interactions is slightly better, but if we make decisions based on AIC or BIC, we should use the model without interactions.
